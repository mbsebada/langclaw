## 2024-05-15 - [CRITICAL] Sensitive Data Exposure in Discord Gateway Logs
**Vulnerability:** The Discord gateway (`langclaw/gateway/discord.py`) logged the first 80 characters of incoming message content (`message.content[:80]`) at the `DEBUG` level during event processing and message dropping.
**Learning:** Logging partial message contents during initial receipt of messages from chat gateways (like Discord) can inadvertently expose sensitive PII, secrets, or confidential user data to application logs, especially since users often use chat interfaces for sensitive workflows. The truncation to 80 characters does not mitigate this risk, as sensitive tokens (like API keys) often fit within that length.
**Prevention:** Avoid logging the actual content of user messages at early gateway entry points. Instead, log metadata such as the length of the message (`len(message.content)`) to aid debugging without compromising data security. Add explicit security comments (`# SECURITY: ...`) to warn future developers against re-introducing content logging.

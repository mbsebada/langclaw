## 2025-03-31 - [Critical] Missing Authentication on Internal Zalo Proxy Service
**Vulnerability:** The internal Zalo Node.js service (`zalo-service`) and its Python proxy in `examples/rentagent_vn/api/routes/zalo.py` were not enforcing authentication via `x-api-key`, which meant anyone who could reach those internal routes could trigger messages or auth functions. Additionally, there was a hardcoded phone number in the outreach sender.
**Learning:** Internal microservices and proxies often lack adequate authentication because they are assumed to be safe behind the perimeter. Furthermore, hardcoded PII/test config can lead to unintended recipients if the codebase isn't using explicit environment variable overrides.
**Prevention:** Always enforce strong service-to-service authentication (e.g., API keys, mTLS) even inside the internal network. Always use environment variable overrides (like `ZALO_PHONE_OVERRIDE`) instead of hardcoding PII/test numbers directly in source files.

## 2024-05-18 - Missing Zalo Service Auth & Hardcoded Phone Override
**Vulnerability:** The Node.js Zalo proxy service lacked API Key authentication for its `/auth` and `/message` endpoints, exposing Zalo operations over the local network. Additionally, the python integration had a hardcoded phone number ("0334663383") which functioned as an insecure target override, causing risks of accidental real-world message delivery or PII leakage.
**Learning:** Proxy services in Node.js need independent authentication (via headers like `x-api-key`) and must employ timing-attack safe string comparison (`crypto.timingSafeEqual`) after length validation to securely handle secrets, instead of relying solely on obscured ports or routing. Also, test overrides should be environment-driven rather than hardcoded in the primary source logic.
**Prevention:** Implement API authentication systematically on all internal services or proxies. Replace any temporary code (like hardcoded test overrides) with environment variables (e.g. `ZALO_PHONE_OVERRIDE`) to prevent accidental commits of PII or unsafe behavior in production.

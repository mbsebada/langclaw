## 2024-04-10 - Missing Authentication & Hardcoded PII in Zalo Integration
**Vulnerability:** The Zalo proxy Node.js service (`zalo-service/index.js`) completely lacked authentication, exposing internal messaging endpoints. Additionally, the FastAPI outreach integration (`api/routes/zalo.py`) had a hardcoded test phone number that could leak PII or send accidental messages.
**Learning:** Services built to wrap external APIs (like zca-js) must have authentication configured by default to prevent unauthorized access. Timing-attack resistant checks (like `crypto.timingSafeEqual`) should be used for API key validation. Furthermore, hardcoded phone numbers in endpoints pose a high risk of accidental messaging and PII leaks.
**Prevention:** Always implement an authentication middleware for wrapper services. Use `crypto.timingSafeEqual` with `Buffer` objects for secure string comparison. Use environment variable overrides (e.g., `ZALO_PHONE_OVERRIDE`) instead of hardcoded numbers for testing, defaulting to the actual user data otherwise.

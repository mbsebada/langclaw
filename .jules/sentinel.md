## 2025-04-15 - Missing Authentication and Hardcoded Information in Internal Services
**Vulnerability:** A missing authentication mechanism was identified for service-to-service communication between the backend API and a Node.js companion service, potentially allowing unauthenticated internal calls if exposed. In addition, an endpoint possessed a hardcoded phone number intended for testing, causing real data (PII) leakage or accidental message dispatch.
**Learning:** Hardcoded testing values left in production code routes will bypass intended system functionality, and auxiliary companion services running alongside main APIs must have authentication to establish trust bounds and limit SSRF risk. Further, service-to-service key checks using `crypto.timingSafeEqual` in Node require equivalent buffer length assertions prior to constant time equality to prevent crashes or partial match exploits.
**Prevention:** Rely on environmental overrides (like `ZALO_PHONE_OVERRIDE`) instead of committing hardcoded constants to test flows. Implement robust middleware leveraging API Keys on all non-health companion service routes to protect against unauthorized service execution.

## 2024-05-24 - Missing Authentication on Internal Microservice
**Vulnerability:** The internal Zalo Node.js service (`examples/rentagent_vn/zalo-service/index.js`) lacked authentication, allowing any internal network traffic to invoke sensitive endpoints (like `/message/send` and `/auth/logout`) without providing credentials. It was assumed that network-level restrictions were sufficient, which violates defense-in-depth principles.
**Learning:** Internal proxy architecture must not assume the internal network is entirely secure. SSRF vulnerabilities or network misconfigurations can expose internal microservices. Even if an endpoint is bound to `localhost`, authentication must be required unless it is explicitly designed to be unauthenticated (e.g., `/health`). In addition, the proxy forwarding to the microservice must provide these credentials.
**Prevention:** Implement strict authentication (e.g., API keys via `x-api-key` headers) for all microservices, verify configuration variables at application startup, and ensure internal proxies pass necessary authentication tokens.

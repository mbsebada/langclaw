## 2024-03-26 - Missing API Authentication for Zalo Service
**Vulnerability:** The internal Zalo Node.js wrapper service (`examples/rentagent_vn/zalo-service/index.js`) did not implement any authentication check for its REST endpoints, despite the proxy endpoint `_proxy_to_zalo` documentation indicating it required API key authentication. A malicious local process or an SSRF vulnerability could have accessed the unauthenticated Node.js service to send arbitrary Zalo messages, check statuses, or initiate/destroy Zalo authentication sessions without any credentials.
**Learning:** Even when microservices are intended to be accessed locally by the main backend, relying solely on network boundaries without proper application-level authentication is a significant security risk, particularly for services handling credentials and outgoing messaging (Zalo wrapper). Additionally, hardcoded test data (like phone numbers) must be thoroughly audited before pushing to main branches, as it could unintentionally expose or target private individuals.
**Prevention:** Always implement strong authentication (such as shared secrets or API keys) on all internal microservices and APIs, even if they are only exposed on localhost. Create tests to simulate unauthorized local requests to ensure they are blocked. Regularly audit code for hardcoded secrets, PII, and testing artifacts before releasing code to production.

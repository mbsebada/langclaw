## 2025-05-01 - [CRITICAL] Fix missing authentication on internal Zalo service
**Vulnerability:** The internal Zalo Node.js wrapper service was exposed to the local network without authentication. The `/message/send` endpoint could be invoked directly, allowing unauthorized sending of SMS messages, or could be leveraged via Server-Side Request Forgery (SSRF) from an attacker to access the internal microservice.
**Learning:** Internal microservices on the local network (like the Zalo service wrapped by Express) must enforce their own authentication layer (such as an API key via middleware), even if they are only intended for access via an internal API proxy.
**Prevention:** Always secure internal Express applications with middleware that validates a shared secret or token (`x-api-key`) provided via environment variables, and ensure proxy applications are configured to transmit this secret securely. Secure the string comparison by comparing byte length and using a timing-safe equality function (e.g., `crypto.timingSafeEqual`) instead of direct string comparison.

## 2024-05-09 - [CRITICAL] Fix hardcoded phone override and missing proxy auth
**Vulnerability:** A hardcoded phone number in the production code meant all test/production Zalo outreach messages were going to the same number `0334663383`, overriding the real landlord's phone. Furthermore, the `_proxy_to_zalo` method was missing API key authentication headers entirely, allowing unauthenticated internal proxy requests.
**Learning:** Hardcoding test phone numbers directly into production API routes is a critical error. The `_proxy_to_zalo` was likely copied from an internal service where API keys were implicitly handled, causing proxy routes to fail when auth was expected in production environments. We must rely on environment configurations to override contact details or pass critical keys in Node.js integrations.
**Prevention:** Implement an environment variable fallback mechanism (e.g. `ZALO_PHONE_OVERRIDE`) instead of modifying the source code for test phone numbers. Review and test all internal API proxy clients to ensure proper authentication (e.g., `ZALO_SERVICE_API_KEY`) is injected during proxy client instantiation.

## 2025-04-02 - [CRITICAL] Fix missing authentication in Zalo service
**Vulnerability:** The Node.js Zalo proxy service lacked authentication middleware, leaving sensitive endpoints (`/auth` and `/message`) exposed to unauthenticated requests. Additionally, an outreach endpoint contained a hardcoded phone number.
**Learning:** Microservice architectures must not rely solely on network segmentation; API key or token-based authentication must be enforced at the service boundary. Furthermore, test credentials/targets should not be hardcoded in production routes; rely on environment variable overrides instead.
**Prevention:** Implement secure-by-default middleware for cross-service communication requiring secrets (e.g., `x-api-key`). In routing logic, use environment overrides (e.g., `ZALO_PHONE_OVERRIDE`) for testing rather than hardcoding PII/target data. Always ensure that authentication middleware fails securely (returning HTTP 500) if required configuration is missing to avoid "fail-open" scenarios.
